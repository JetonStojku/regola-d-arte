# Regola Mobile Nav

Before editing navigation:

1. Read `.codex/rules/regola-layout.md`.
2. Keep mobile header to logo, compact booking CTA, and menu button.
3. Put language choices inside the menu, not beside every primary item.
4. Keep desktop navigation, language switcher, CTA, and social links visually separated.
5. Keep persistent social links outside the header on mobile.

Before finishing:

- Verify menu controls are keyboard reachable.
- Verify language links are labeled and separated from primary nav.
- Verify floating social links do not block footer legal anchors or main CTAs.
- Run `npm run build` in `website/` when code changed.
