# Layout refresh audit

## Goal

Document the current layout problems and the target layout patterns before making future visual edits.

## Required rules / skills

- Read `.codex/rules/regola-layout.md`.
- Use `.codex/skills/regola-layout-review/SKILL.md`.

## Implementation plan

1. Review header, footer, home, about, services, service detail, and contact pages.
2. Identify dense navigation, visually merged controls, unbalanced columns, invisible social links, and text-heavy areas.
3. Record which sections need real photos, placeholders, or content confirmation.
4. Keep before/after clinical photos as placeholders unless explicitly approved.

## Acceptance criteria

- [ ] Layout issues are described with page and section references.
- [ ] Target patterns are clear enough for implementation.
- [ ] No unapproved clinic facts or clinical photos are introduced.

## Verification

Compare desktop and mobile screenshots for `/it/`, `/en/`, services, and contact pages.
