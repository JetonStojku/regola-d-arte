# Codex rules and skills upkeep

## Goal

Keep project-local Codex rules and skills aligned with the owner’s layout preferences.

## Required rules / skills

- Read `.codex/rules/regola-layout.md`.
- Read `.codex/rules/regola-images.md`.

## Implementation plan

1. Update rule files when the owner identifies a recurring mistake.
2. Update the relevant skill when a workflow needs a repeatable checklist.
3. Keep skills concise and task-specific.
4. Reference the rules and skills from new task files.

## Acceptance criteria

- [ ] New preferences are captured in rules or skills.
- [ ] Task files tell future Codex runs which guidance to read.
- [ ] Guidance remains short enough to be used in context.

## Verification

Review `.codex/rules/`, `.codex/skills/`, and task references after every preference update.
